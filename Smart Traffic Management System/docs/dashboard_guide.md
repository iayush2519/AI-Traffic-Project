# Extended Detail: Nexus Traffic AI Interface Guide 

This document offers a deeply detailed, component-by-component analysis of the **Dashboard**. It explicitly describes what data feeds into each card, what actually happens under the hood when traffic events occur, and the exact purpose each system serves for the operator.

---

## 1. Overview Tab (The Command Center)

The target of the Overview tab is to unify live telemetry streaming via WebSockets (`ws://localhost:8000/ws/traffic`) into a singular, digestible screen. 

### A. Core Statistics (KPI Cards)
These four cards at the top sit directly over the raw data stream, recalculating their totals every 5 seconds as the backend runs its simulation loop.
* **Active Vehicles:** What happens: The frontend iterates over all 20 intersections arriving in the WebSocket payload and sums up the `vehicle_count`. In the demo screenshot (e.g. 946), this is the physical number of cars sitting in queues globally.
* **Avg Wait Time:** What happens: Calculates the mathematical average of the `waiting_time` attribute across all nodes. 
* **Network Efficiency & Severe Alerts:** What happens: As intersections cross a threshold density (over 75%), they are tagged as `severe`. The Severe Alerts card instantly flashes Red and displays the exact count of intersections currently collapsing into gridlock.

### B. Live Traffic Map (Geospatial View)
* **What happens:** The application uses React-Leaflet localized strictly over Los Angeles bounds. The system receives latitude (`lat`) and longitude (`lon`) from the WebSocket and plots standard markers.
* **Colors & Interaction:** The markers dynamically shift colors based on the `congestion_label` (Green=low, Yellow=medium, Orange=high, Red=severe). 
* **When you click:** A specialized popup surfaces displaying the specific Intersection Number, current raw Speed (e.g., 35mph), Density (e.g., 85%), and crucially, the **15-minute future prediction** generated by the AI block.

### C. System Watchdog Alerts
* **What it does:** It is an automated notification stack.
* **What happens behind the scenes:** Inside the custom `useTrafficData` React hook, every time a new WebSocket message arrives, it filters for intersections labeled `severe` or predicted to be `severe` in 15 minutes. It generates a unique timestamped alert and pushes it into the UI.
* **Interactivity:** The system keeps a maximum of the 3 most recent alerts to prevent screen clutter. An operator can explicitly click the `X` to dismiss an alert if they have resolved the issue.

### D. Adaptive Signal Controller
* **What it does:** Surfaces the literal timer-splits running traffic lights in real-time.
* **What happens:** It limits the view to only the top 5 most heavily congested intersections (sorting by `traffic_density`). For an Intersection (e.g., "Int 3"), it visually draws the precise seconds assigned to the North-South (NS) green light versus the East-West (EW) green light.
* **The Progress Bar:** Beneath the timers is a dynamically rendered CSS multi-color progress bar. This represents the total `cycle_time` (e.g., 90s) visually, showing the exact percentage ratio the AI has decided to give to the NS phase versus the EW phase.

### E. Analytics Charts (Bottom Row)
* **Top Congested Intersections:** A Recharts bar-graph constantly recalculating the top ranked worst intersections by density, mapping them horizontally colored from yellow to red. 
* **Network Average Speed:** A historical Recharts local-graph that tracks exactly how network speeds change per minute.

## 2. Analytics Tab (Deep Model Telemetry)

When an administrator clicks the Analytics sidebar icon, the UI completely swaps to view system health rather than traffic health.

### A. Model Performance Card
* **What happens:** Based on exactly which Model is selected in the Settings tab (e.g., "LSTM (V2)" or "XGBoost (V1)"), this card reads from the Global React Context and actively updates its title and numbers. It displays hard metrics like ROC-AUC (0.915) and 30-minute accuracy (e.g. 94.2%), proving that the AI is technically sound.

### B. Data Pipeline & Node Status
* **What happens:** Displays metrics like Processing Latency (42ms) and Ingestion Rate. In an actual multi-node docker deployment, if latency spikes over 500ms, the operator inherently knows the system is lagging behind reality and the map data might be stale.

---

## 3. Signals Tab (The Engineering Audit)

* **What it does:** It completely drops all visual abstractions (charts/maps) and presents a raw, 11-column spreadsheet of the telemetry matrix.
* **What happens:** The table extracts all the un-abbreviated figures received from the WebSocket: exact `road_class` (Local vs Arterial), `volume`, `queue_length`, current Delay, and precise predictions for 15 minutes out.
* **Why this format:** Planners and Traffic engineers require tabular formats during legal or safety audits. If an accident occurs, engineers use this exact tabular view to track what the AI’s exact NS/EW splits were at any given second.

## 4. Settings Tab (Safety Rails & Configuration)

The Settings tab is functionally the most critical interface because it links the "Human Operator" directly into the loop to constrain the AI. 

* **What happens when you change settings:** All modifications here are stored in the `SettingsContext.js` global state, meaning if you update a setting and swap back to the Map, your modifications remain intact.

### A. Model Strategy Toggles
* **What it does:** Allows the administrator to hot-swap the internal prediction engine from Long Short-Term Memory Neural Networks (LSTM) to Decision Trees (XGBoost) or strict basic math rules (Rule-Based Fallback). 

### B. Controller Safety Bounds (The "Kill-switches")
* **Auto-Apply Adaptive Timings:** Enables or disables the core functionally of the project. If toggled OFF, the AI will only *suggest* timings in the dashboard, but will cease sending the API requests to actually change physical lights out on the street.
* **Emergency Corridor Override:** A simple toggle that, when enabled, forces the AI to prioritize huge green-light percentages to core arterials, generally used when ambulances or massive city-evacuations are occurring.
* **Minimum Green Slider:** Operators drag exactly how low the AI is permitted to drop a green light (e.g., 5 seconds to 25 seconds). **Why this matters:** The AI might realize an intersection is empty and want to give a 1-second green light. However, real-world physics dictates a 1-second light causes massive confusion and T-bone car crashes. This slider physically blocks the AI from dropping below standard human-reaction times (like 12s). 
* **Save Logic:** Clicking Save throws a 2-second visual confirmation letting the user know the configurations were successfully recorded into the global state context.
